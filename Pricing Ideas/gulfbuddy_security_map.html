🚨 Current Security Gaps (fix before go-live)
🔑
API Key Exposed in Frontend JavaScript
Supabase anon key is hardcoded in every HTML file — visible to anyone who views page source. Currently safe because RLS (Row Level Security) is not fully configured. Must fix before launch.
Critical
🗄️
Supabase RLS (Row Level Security) Not Configured
Right now anyone can read, write, or delete ANY row in your database directly via API. No authentication required. A simple API call from anyone's browser can wipe all listings.
Critical
💉
No Input Sanitisation
Post forms accept any text including HTML/JavaScript. Someone can post a listing with script tags that execute on other users' browsers (XSS — Cross Site Scripting). All form inputs need sanitising before saving to DB.
Critical
🌐
No HTTPS / SSL Certificate
Currently localhost — no issue now. But when live on gulfhabibi.com, must have SSL certificate. Without it browsers show "Not Secure" warning and Google penalises your ranking.
Fix at launch
⚡
No Rate Limiting
Anyone can spam your post form or search endpoint thousands of times per second. Can crash the server or exhaust your Supabase free tier rows in minutes. Need rate limiting on all API calls.
Important
📁
No File Upload Validation
Images.js compresses images — good. But doesn't check if the file is actually an image. Someone could upload a disguised executable or malicious file. Need MIME type + file signature validation.
Important
🔐
Password Gate (index.html)
Current password gate (gulf2024) protects the preview URL. Basic but effective for localhost testing phase.
OK for now
🖼️
Image Compression (images.js)
Already built — strips EXIF metadata (GPS location data) from photos before upload. Protects seller privacy. Max size limits prevent file bomb attacks.
Done
